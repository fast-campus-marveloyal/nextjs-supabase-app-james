🗂️ 린 캔버스 (Lean Canvas)

모임 이벤트 관리 웹 MVP · docs/PRD.md v0.2 기반 · 작성일 2026-07-04

1문제

(Problem)

  • 공지사항을 카톡 단톡방에 흩뿌려 전달 — 묻히고 검색 안 됨
  • 참여 인원 파악을 "손✋" 방식으로 수동 집계
  • 카풀 수요/공급을 단톡방 텍스트로 매칭 — 스크롤로 추적
  • 대관료·회식비를 인원수로 나눠 정산, 입금 여부를 일일이 확인

4해결책

(Solution)

  • 이벤트 생성 + 공유 링크 자동 발급
  • 공지 CRUD + 상단 고정
  • 참여자 명단 관리(주최자 직접 입력)
  • 카풀 등록 → 탑승 신청 → 승인/거절 (수동 매칭)
  • 정산 총액 ÷ 인원 자동 계산 + 입금 체크리스트

3가치 제안

(Unique Value Proposition)

  • 카톡 단톡방을 대체하는, 주최자 혼자 쓰는 모임 운영 올인원 도구
  • 공지·명단·카풀·정산을 한 화면에서 관리
  • 참여자는 회원가입 없이 링크 하나로 모든 현황 확인

5경쟁우위

(Unfair Advantage)

⚠️ PRD 미기재 — 추정
  • 실사용자(주최자 본인)가 곧 개발자 — 실제 운영 마찰을 빠르게 제품에 반영
  • "소모임 정산/카풀"이라는 좁고 구체적인 니치에 집중 (범용 이벤트 툴 대비 학습비용 낮음)

2고객 세그먼트

(Customer Segments)

  • 주 타겟: 수영·헬스·취미 소모임을 정기 운영하는 주최자 1인 (비개발자)
  • 얼리어답터: 이미 3회 이상 정기 모임을 운영해 카톡방 관리 피로가 누적된 주최자
  • 보조 사용자: 참여자(비계정) — 공유 링크로 조회 및 카풀 신청만

8주요 지표

(Key Metrics)

  • 생성된 이벤트 수 / 활성 이벤트 수
  • 이벤트당 등록 참여자 수
  • 공유 링크 조회수(참여자 방문)
  • 정산 완료율 (입금 체크 비율)
  • 카풀 신청 → 승인 처리율

9채널

(Channels)

  • 카카오톡 공유 링크 (핵심 유입 채널 — 참여자가 곧 잠재 주최자)
  • 동호회 오픈채팅방 / 커뮤니티 카페
  • 기존 주최자의 입소문 추천

7비용 구조

(Cost Structure)

  • Supabase 프로젝트 비용 (DB·Auth, 사용량 기반)
  • Vercel 등 프론트엔드 호스팅 비용
  • 도메인 비용
  • (향후) 카카오 알림톡 발송비, PG 결제 수수료 — 현재 MVP 범위 밖
