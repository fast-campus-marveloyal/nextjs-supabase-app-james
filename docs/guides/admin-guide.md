# 관리자 사용 가이드

견적서 관리 시스템 관리자 기능 사용 방법을 안내합니다.

## 관리자 로그인

1. 브라우저에서 `/admin-login` 경로로 접속합니다.
   - 로컬: `http://localhost:3000/admin-login`
   - 프로덕션: `https://your-app.vercel.app/admin-login`
2. 비밀번호 입력창에 `.env.local`의 `ADMIN_PASSWORD` 값을 입력합니다.
3. **로그인** 버튼을 클릭합니다.
4. 로그인 성공 시 `/admin` 대시보드로 자동 이동합니다.
5. 세션은 7일간 유지됩니다. 만료 후에는 재로그인이 필요합니다.

> 비밀번호가 틀린 경우 화면 우하단에 "비밀번호가 일치하지 않습니다" 에러 토스트가 표시됩니다.

---

## 견적서 목록 조회

로그인 후 좌측 사이드바에서 **견적서 목록** 메뉴를 클릭하거나 `/admin/invoices`로 직접 이동합니다.

### 검색

1. 상단 검색창에 **클라이언트명** 또는 **견적서 번호**를 입력합니다.
2. `Enter` 키를 누르거나 검색 아이콘을 클릭합니다.
3. URL에 `?query=검색어` 파라미터가 추가되며 결과가 필터링됩니다.
4. 검색어를 지우려면 검색창 우측 X 버튼을 클릭합니다.

### 필터

1. 검색창 옆 **필터** 버튼을 클릭합니다.
2. 드롭다운에서 상태를 선택합니다.
   - **전체**: 모든 견적서 표시
   - **대기**: 클라이언트 검토 대기 중인 견적서
   - **승인**: 클라이언트가 승인한 견적서
   - **거절**: 클라이언트가 거절한 견적서
3. 발행일 시작/종료 날짜로 기간 필터를 추가할 수 있습니다.
4. **필터 적용** 버튼을 클릭합니다.
5. 필터가 적용된 경우 버튼에 **숫자 배지**(예: `필터 1`)가 표시됩니다.

### 정렬

- **발행일** 컬럼 헤더 클릭 → 발행일 기준 오름차순/내림차순 정렬
- **총액** 컬럼 헤더 클릭 → 총액 기준 오름차순/내림차순 정렬

### 페이지네이션

- 목록 하단의 **이전** / **다음** 버튼으로 페이지를 이동합니다.
- 한 페이지당 기본 10개의 견적서가 표시됩니다.

## 링크 복사 및 공유

클라이언트에게 견적서를 공유하는 방법입니다.

1. 견적서 목록에서 공유할 견적서 행의 **링크 복사** 버튼을 클릭합니다.
2. 클립보드에 견적서 고유 URL이 복사됩니다.
   - 형식: `https://your-app.vercel.app/invoice/{Notion_페이지_ID}`
3. 복사된 URL을 이메일, 메신저 등으로 클라이언트에게 전달합니다.
4. 클라이언트는 별도 로그인 없이 URL로 견적서를 조회하고 PDF를 다운로드할 수 있습니다.

> **공유** 버튼을 클릭하면 브라우저 공유 메뉴(Web Share API)가 열립니다. 지원하지 않는 환경에서는 링크 복사로 대체됩니다.

## ADMIN_PASSWORD 변경 방법

관리자 비밀번호를 변경하려면 다음 단계를 따릅니다.

### 로컬 개발 환경

1. 프로젝트 루트의 `.env.local` 파일을 엽니다.
2. `ADMIN_PASSWORD=기존비밀번호` 값을 새 비밀번호로 수정합니다.
   ```
   ADMIN_PASSWORD=새로운비밀번호
   ```
3. 개발 서버를 재시작합니다.
   ```bash
   # 기존 서버 종료 후
   npm run dev
   ```

### Vercel 프로덕션 환경

1. [Vercel 대시보드](https://vercel.com/dashboard)에서 해당 프로젝트로 이동합니다.
2. **Settings → Environment Variables**로 이동합니다.
3. `ADMIN_PASSWORD` 항목을 찾아 새 값으로 수정합니다.
4. **Redeploy**를 실행하여 변경사항을 반영합니다.

> 비밀번호 변경 후 기존 세션은 만료(최대 7일)될 때까지 유지됩니다. 즉시 무효화가 필요하다면 `SESSION_SECRET`도 함께 변경하고 재배포합니다.

## 로그아웃

관리자 화면 우상단의 **로그아웃** 버튼을 클릭하면 세션이 삭제되고 로그인 페이지로 이동합니다.
